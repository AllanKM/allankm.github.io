Build vs buy for AI capabilities

Vendors ship impressive assistants. Building in-house feels like control. Both fail when nobody maps the capability to competitive advantage and exit options.

Decide on capability, not logo

Buy commodity: generic summarization, coding assist, standard OCR. Build or deeply configure where underwriting logic, claims nuance, or customer data advantage lives.

Insist on exportable evaluation data, portable prompts or graphs where possible, and clear data-use terms. Soft lock-in starts as convenience.

Price the whole system

Include integration, monitoring, change management, and model churn. A cheap API with expensive human cleanup is not cheap.
