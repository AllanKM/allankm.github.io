Architecture review when half the code was generated

Classic review hunted style and cleverness. With AI in the loop, volume rises and local quality looks fine while structural mistakes hide in plain sight: new coupling, silent schema drift, duplicated domain logic under new names.

Ask different questions

Does this change respect the capability boundary? Which invariants still hold? What happens when the model or library version moves? Is there an exit path if the AI-assisted path stalls?

Demand diagrams or decision notes for anything that crosses trust zones, persists new entities, or alters customer-facing behaviour. Diff size is not a severity score.

Make review scalable

Automate the boring checks. Reserve human review for ownership, risk, and fit to the target operating model. Architecture review is still the agreement between business and IT—generation does not waive it.
